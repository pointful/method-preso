Software Development Method = Process + Practices

Our Agile Process

Scrum, Adapted to size of team and duration of project

  • Product Backlog as Epics and Stories
    • estimated in complexity points
  • Sprint Backlog as Tasks
    • estimated in time
  • Normally two-week Sprints
  • Client-facing demos every Sprint
  • Internal retrospective every Sprint

Our Agile Practices

Various, many from Extreme Programming (XP)

  • User Stories / Use Cases
  • Test Driven Development
    • Unit + Acceptance, Integration, Performance, Security
  • Continuous Integration
  • Merciless Refactoring
  • Collective Code Ownership
  • Project Coding Conventions
